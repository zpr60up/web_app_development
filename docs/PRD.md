# 個人記帳系統 PRD

## 1. 專案概述
- **背景與動機**：為個人打造一款彈性高且符合實際現金流週期（如發薪日）的專屬記帳系統，透過自訂記帳起始日與分類，清楚掌握從計月起始日至當下的收支情況，並於月底自動結算整月使用狀況。
- **目標用戶**：需要管理個人財務的使用者（我）。
- **核心價值主張**：跳脫傳統日曆月的記帳限制，支援**自訂月起始日**與**高度自訂化的收支分類**，搭配直覺的當下與月末財務報表，更貼近個人的真實生活週期。

## 2. 功能需求
1. **收入介面**
   - 作為 **使用者**，我希望 **能在專屬介面中記錄一筆收入（包含金額、分類、日期、備註等）**，以便 **有系統地追蹤資金來源**。
2. **支出介面**
   - 作為 **使用者**，我希望 **能在專屬介面中記錄一筆支出（包含金額、分類、日期、備註等）**，以便 **記錄並掌握日常開銷的流向**。
3. **當下財務報表**
   - 作為 **使用者**，我希望 **能從一個介面中直觀看到「自本月起始日到現在」的累積收支情形與目前可用餘額**，以便 **即時衡量當下的財務健康度**。
4. **月末財務報表**
   - 作為 **使用者**，我希望 **能在每個月循環結束時自動產出本月的總結報表**，以便 **回顧整個月各分類的金錢使用狀況與佔比**。
5. **自訂月起始日**
   - 作為 **使用者**，我希望 **能自訂每個月計算帳務的起始日期（例如每月 5 號發薪日）**，以便 **讓記帳週期完全符合我的實際財務週期**。
6. **自訂收支分類（含基本類型）**
   - 作為 **使用者**，我希望 **除了系統預設的基本收支類型外，還能自由新增、修改或刪除分類項目**，以便 **應對我個人多樣化且客製化的記帳需求**。

## 3. 非功能需求
- **技術限制**：
  - 整體架構必須使用 **Flask + Jinja2 + SQLite** 進行開發。
- **效能與安全考量**：
  - 由於是單人使用，首要考量確保 SQLite 的讀寫與各類日期區間（尤其是自訂起始日）查詢的高效性。
  - 需要防範基本的手動輸入錯誤（如負數金額、空白標題）以及防範 XSS / SQL Injection 等常見漏洞。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 收入及支出記錄介面（支援自訂收支分類）。
  - 自訂月起始日設定（設定後影響全部報表區間）。
  - 當下財務報表（顯示自起始日至今日的總覽）。
- **Should Have (應該有)**：
  - 月末財務報表（依據起始日產出區間總結）。
  - 可以修改或刪除過往記錄。
- **Nice to Have (渴望有)**：
  - 財務報表圖表呈現（透過 Chart.js 或類似套件畫出圓餅圖、長條圖）。
  - 密碼鎖或簡單的單一帳號登入機制。
